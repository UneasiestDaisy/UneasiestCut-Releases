Privacy

What Stays Private

This site publishes installers, version metadata, and release notes only. The main ClipForge Studio source repository stays private.

Public installers can still be inspected, so secrets and internal-only tooling are never treated as safe just because the source repo is private.

Download Visibility

Can We Tell If Anyone Downloads It?

Yes, at a basic level. The installer is hosted as a GitHub release asset, so GitHub provides per-asset download counts on the release page.

That tells you how many times the file was downloaded, not who installed it. If you want deeper attribution later, add site analytics for button clicks or a small redirect service in front of the asset.

FAQ

Update Flow

Do users need a GitHub account?

No. This public site and public release lane are meant to remove GitHub account friction for installs and updates.

Does this expose the private repo?

No. The public lane hosts binaries and release metadata, while the working source repo stays private.

Will updates be version-based?

Yes. New releases should move the app version forward so installed copies can detect the update cleanly.

Can installs be tracked precisely today?

Not from this static page alone. Right now the clean signal is GitHub asset download counts, with optional site-click analytics or a redirect layer if deeper tracking becomes worth it.
